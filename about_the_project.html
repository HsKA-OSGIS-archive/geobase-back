Our Project is Exploring (OGC (Open Geospatial Consortium) API) with Geoserver and OpenLayers (based on BfS (the Bundesamt für Strahlenschutz.) radiological OGC services).

The result of a project for the Open Source GIS course during the Winter 2023-2024 semester as part of the Geomatics MSc, run by the Karlsruhe University of Applied Sciences has publishe on GitHUB.

The project as a whole aimed to explore the new features and possibilities offered by the new OGC API framework, a new standard based on OpenAPI which is succeeding the new OGS API standard.

Example data was obtained from an OWS service of Radiological Data hosted by the German Bundesamt fuer Strahlenschutz (BfS) the new OGC API-Feature standard in place of the WFS standard.

Thus, this webpage and the associated open documentation can provide a model for future implementations of this new OGC API-Feature standard. For further information about the new OGC API-Feature standard, follow the link on the OGC API Documentation menu.

Our project repository has publish on GitHub Repository.

This Webpage displays hourly updated gamma doese rate from measurement stations in Germany. The data source is the the Webpage of the Bundesamt für Strahlenschutz.

The Benifit of ths Website is, that our data is projected via the new OGC API-Feature standard in place of the WFS standard. Thus, this webpage and the associated open documentation can provide a model for future implementations of this new OGC Standard. For further information about the new OGC API standard, follow the link on the top menu bar.

Our goal: Explore the current possibilities of OGC API standards by rehosting (or relaying) publicly available OWS features and displaying them via OpenLayers.

GOSolutions Research: A fictional company, GOSolutions, conducted research on OGC API, the new standard for Web services of the OGC.

Dynamic OGC API Development: OGC API development is a very dynamic field, so it's necessary to compare various OGC API implementations in order to update the status quo on this topic.

Comparison Process: Therefore, it is necessary to set up different Open Source products that serve OGC API and compare these in a small web frontend.

  • Rehost publicly available OWS features
  • Display features via OpenLayers
  • Conduct research on OGC API standards
  • Compare various OGC API implementations
  • Set up different Open Source products for comparison

Target of our project: Migrating from old API to new API
